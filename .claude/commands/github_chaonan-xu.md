GitHubリポジトリ https://github.com/satomaru0512/chaonan-xu.git に、指定したディレクトリやファイルをコミット・プッシュするスキルです。

以下の手順で実行してください：

1. ユーザーに以下を確認する：
   - コミット対象のファイルまたはディレクトリ（未指定の場合は変更済みファイル一覧を表示して選ばせる）
   - プッシュ先のブランチ（未指定の場合は現在のブランチを使用）
   - リモートリポジトリは常に https://github.com/satomaru0512/chaonan-xu.git を使用する

2. `git status` を実行して現在の変更状況を確認・表示する

3. 指定されたファイル/ディレクトリを `git diff` または `git diff --cached` で差分を確認し、変更内容を把握する

4. 変更内容を分析して、以下の形式でコミットメッセージを自動生成する：
   - 1行目: 変更の要約（例: "feat: ログイン機能を追加", "fix: nullポインタエラーを修正"）
   - 空行
   - 以降: 変更の詳細（何を変更したか、なぜ変更したか）

5. 生成したコミットメッセージをユーザーに提示し、確認または修正を求める

6. ユーザーが承認したら以下を実行する：
   - `git add <指定ファイル/ディレクトリ>`
   - `git commit -m "<コミットメッセージ>"`
   - `git push <リモート> <ブランチ>`

7. 完了後、コミットハッシュとプッシュ先URLをユーザーに報告する

注意事項：
- コミットメッセージには必ず変更内容の説明を含めること
- .env や認証情報などの機密ファイルが含まれていないか確認すること
- 強制プッシュ（--force）はユーザーが明示的に要求した場合のみ実行すること
